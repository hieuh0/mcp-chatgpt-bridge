Problem & solution

Hỏi thêm một AI — nhanh hơn hỏi người, nếu làm đúng cáchAsking another AI — faster than asking a human, if done right

Khi Claude Code gặp điểm quyết định quan trọng — pattern A hay B? cách xử lý lỗi này đúng không? — việc tham vấn AI khác chỉ có giá trị khi ba điều kiện được đảm bảo: When Claude Code hits a real decision point — pattern A or B? is this error handling correct? — consulting another AI only helps if three conditions hold:

Con người vẫn xem lại được đã hỏi gì, được khuyên gì — qua Telegram/Slack hoặc dashboard.The human keeps a full record of what was asked and recommended — via Telegram/Slack or the dashboard.

Xây trên chuẩn mở Model Context Protocol của Anthropic[1] — AI client kết nối chuẩn hoá tới data/tool server, không cần tích hợp riêng lẻ. Built on Anthropic's open Model Context Protocol standard[1] — AI clients connect to data/tool servers through one standardized interface, no custom integrations.

Security by design

Chặn từ gốc, không chặn qua loaClosed at the root, not patched over

+
Dashboard chỉ bind 127.0.0.1 — không remote/multi-user, đúng use-case cá nhân local.Dashboard binds 127.0.0.1 only — no remote/multi-user, matches its local single-user use case.
+
API keys lưu trong SQLite (data.sqlite, quyền 0o600) — không bao giờ log ra plain text.API keys stored in SQLite (data.sqlite, mode 0o600) — never logged in plain text.
+
Route fetch-models có 2 lớp riêng: chặn request không Origin, và pin IP đã resolve để chặn DNS-rebinding TOCTOU (loopback/link-local/169.254.169.254/private/CGNAT).The fetch-models route adds two extra guards: rejects requests with no Origin, and pins the resolved IP to close a DNS-rebinding TOCTOU gap (loopback/link-local/169.254.169.254/private/CGNAT).
⟶ Thiết kế này ra đời sau một vòng red-team review đối kháng trước khi code — bản đầu (chỉ validateBaseURL + same-origin lỏng) không thực sự chặn được SSRF/rò rỉ credential. ⟶ This design emerged from an adversarial red-team review before implementation — the original (just validateBaseURL + a lenient same-origin check) didn't actually close the SSRF/credential-exfiltration risk.
